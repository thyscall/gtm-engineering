# GTM Engineering Portfolio

Production-ready scripts and automated systems designed to remove manual friction from the revenue engine. Their goal is to bridge the gap between raw data and revenue by automating lead routing, intent signal detection, CRM hygiene and more. 


## Core Intent
Revenue teams struggle with fragmented data and manual triage. This portfolio has technical solutions that act as an engine that handles the dirty data engineering for Sales and Marketing teams so they can focus on high-value human engagement.


## Functional Workflows
The repository is organized by project intentions:
* **Signal Routing:** Automated monitors that scan for market triggers and push alerts to Slack or CRM webhooks.
* **Data Engineering:** Scripts for ICP scoring, lead de-duplication, and multi-source "waterfall" enrichment.
* **Operational Guardrails:** Systems that enforce SLAs on lead response times and automate deal-room provisioning.


## Architecture & Stack
The projects will utilize a modular architecture with a `shared/` directory for reusable API logic.
* **Languages:** Python 3.10+
* **Core Stack:** Pandas, Requests, and more soon. 
* **Integrations:** HubSpot, Salesforce, Apollo, Clay, Slack, and various REST APIs.
